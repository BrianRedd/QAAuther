Editing QA Authenticator Data:

Click on Column Headers to sort table by appropriate column.

Click to add and edit a new record (new record will appear at top of list).

Click to edit a row.

Click when done. Edited rows will appear in blue (even if no data was changed).

Click to remove a row (you will be prompted to confirm before row is deleted).

If you accidentally delete a row you shouldn't have, refresh the page (hit F5). NOTE: Doing so will erase ALL unsaved changes applied to the data.

Click to create a new clone row and edit it.

Click to open XML content in a selectable format; this content should then be saved to the QAAutherData.xml file.

To filter displayed rows, use the filter fields; choose the column from the drop down then enter the text in the text field.
[Apply Filter] will remove all items from the display that do not match your criteria. You can apply multiple filters.
[Clear Filter] will clear all filters.

Special Rules:

  1. Short URL, such as http://[domain]/[skin_name]
  2. RSSFeed located under http://demo.olivesoftare.com/[parent]/feeds/[skin_name]/manifest.xml
  3. Short URL, such as http://[domain]/, with no Skin Name
  4. RSSFeed located under http://demo.olivesoftare.com/repository/[repos]/[yyyy]/[mm]/[dd]/res/channels/kindlefeed/manifest.xml
  5. External site with added index link
  6. Uses "Default" virtual directory for ATS, but with unique ProjectId
  7. Uses "Default" virtual directory for APD2.x (Low Level URL) instead of "Daily"
  8. Time Traveler application (default.aspx replaced with defaultTT.aspx
  9. Skin is HTTPS (but not using digital2.olivesoftware.com)
